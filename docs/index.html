          Redis — atomic ops
        

        

          
# Atomic click counter (no DB write on redirect path)

          
127.0.0.1:6379> INCR click_counter:xKj9mQp

          
(integer) 847

          

          
# Unique click dedup — SET NX with 1h TTL

          
127.0.0.1:6379> SET unique_click:xKj9mQp:a1b2c3 1 EX 3600 NX

          
OK  # first click — unique

          
127.0.0.1:6379> SET unique_click:xKj9mQp:a1b2c3 1 EX 3600 NX

          
(nil)  # same IP within 1h — not unique

          

          
# Atomic drain (Lua) — GET + DEL in one operation

          
127.0.0.1:6379> EVAL "local v=redis.call('GET',KEYS[1]) ..."

          
(integer) 847  # flushed to MySQL, counter reset

        

      

    

  


  
  

    

      ⬡
      Architecture Highlights
    

    

      

        

          
⚡

          
Cache-First Redirect

        

        
Redis lookup on every redirect. DB query only on cache miss. <5ms on hit, <25ms on miss. Zero blocking I/O on the hot path.

      

      

        

          
🌍

          
Async Geo-IP

        

        
Inline geo-IP adds 150–300ms to every redirect. We dispatch a queued job instead — the 301 fires before enrichment even starts.

      

      

        

          
🔢

          
Atomic Counters

        

        
Redis INCR on every click (~1µs). Lua script drains counters to MySQL every minute. Zero DB writes on the redirect path.

      

      

        

          
🛡

          
Rate Limiting

        

        
Separate limiters: 10/min on shorten (prevents spam), 120/min on redirect (prevents enumeration). Per-user when authenticated.

      

      

        

          
🧪

          
Full Test Suite

        

        
Unit tests mock all I/O. Feature tests use SQLite :memory: + real Redis DB 15. Http::fake() for geo-IP, Queue::fake() for jobs.

      

      

        

          
📋

          
OpenAPI Docs

        

        
Full Swagger UI auto-generated from PHPDoc annotations. Every endpoint documented with request/response schemas and auth requirements.
